# 前后端对接说明

## 环境配置
- 前端模式：`.env` 中 `VITE_ACCESS_MODE=backend`，开发代理转发到后端（`VITE_API_PROXY_URL=http://127.0.0.1:9123`）。
- 后端默认监听：`http://127.0.0.1:9123`。如端口被占用，请先结束旧进程后再启动。
- 启动命令：
  - 后端：`cd backend && cargo run -p api`
  - 前端：`cd frontend && pnpm dev`

## 接口一览
- 认证
  - `POST /api/auth/login`：登录，入参 `{ userName, password }`。
  - `POST /api/auth/register`：注册，入参 `{ userName, password, email? }`。
  - `POST /api/auth/forget-password`：重置密码，入参 `{ userName, newPassword? }`（默认 123456）。
  - `GET /api/user/info`：获取当前登录用户信息（需 Authorization）。
- 系统管理
  - `GET /api/user/list`：用户列表，支持分页和搜索字段（id/userName/userGender/userPhone/userEmail/status）。
  - `POST /api/user`：新增用户，入参 `{ userName, phone?, email?, status?, roleCodes? }`（默认密码 123456）。
  - `PUT /api/user`：更新用户，入参 `{ id, phone?, email?, status?, roleCodes? }`。
  - `DELETE /api/user/{id}`：删除用户。
  - `GET /api/role/list`：角色列表，支持分页和搜索字段（roleId/roleName/roleCode/description/enabled）。
  - `POST /api/role`：新增角色。
  - `PUT /api/role`：更新角色。
  - `DELETE /api/role/{id}`：删除角色。
  - `GET /api/v3/system/menus/simple`：菜单树，返回与前端路由一致的结构（组件路径、roles、authList 均已对齐）。

## 数据返回约定
- 统一响应：`{ code, msg, data }`，`code=200` 表示成功，401 触发前端自动登出。
- 用户信息：返回真实角色代码（R_SUPER/R_ADMIN/R_USER）和按钮权限。
- 菜单数据：二级子路由使用相对 path（如 `console`、`user`），组件路径如 `/dashboard/console`，避免前端路由校验告警。

## 前端对接点
- 登录/注册/忘记密码页面已接通真实接口（`frontend/src/views/auth/...`）。
- 菜单加载走后端模式，RouteRegistry 动态注册；后台接口返回的 routes 已与前端视图路径一致。
- 系统管理页面（用户/角色）增删改已接通上述接口；菜单管理仍为只读，后续如需持久化需补充菜单表与 CRUD。

## 注意事项
- 角色与按钮权限目前由后端按用户角色派生，若需更细粒度控制可扩展字段。
- 迁移简化为单个初始化脚本（用户/角色/关联和基础种子数据），目前不创建菜单表。
