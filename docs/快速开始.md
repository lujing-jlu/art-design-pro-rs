# 快速开始指南

## 环境要求

### 后端
- Rust 1.70+ (推荐使用 rustup 安装)
- SQLite 3 (或 PostgreSQL/MySQL)

### 前端
- Node.js 20.19.0+
- pnpm 8.8.0+

## 安装步骤

### 1. 克隆项目

```bash
git clone <repository-url>
cd art-design-pro-rs
```

### 2. 启动后端

```bash
cd backend

# 首次运行会自动下载依赖并编译
cargo run -p api

# 或使用 release 模式（更快）
cargo run -p api --release
```

后端服务将在 `http://127.0.0.1:9123` 启动。

**首次启动时会自动**:
- 创建数据库文件 `backend/data.db`
- 执行数据库初始化（建表 + 种子）
- 创建初始数据（3 个测试账号）

### 3. 启动前端

打开新的终端窗口：

```bash
cd frontend

# 安装依赖（首次运行）
pnpm install

# 启动开发服务器
pnpm dev
```

前端服务将在 `http://localhost:4000` 启动，浏览器会自动打开。

## API 测试

### 使用测试脚本

后端提供了两个测试脚本：

```bash
cd backend

# 测试基础接口（登录、用户、角色、菜单）
./test_api.sh

# 测试用户中心接口（个人信息、密码、头像）
./test_user_center.sh
```

**注意**: 测试脚本需要 `jq` 工具来格式化 JSON 输出。

安装 jq:
```bash
# macOS
brew install jq

# Ubuntu/Debian
sudo apt-get install jq

# CentOS/RHEL
sudo yum install jq
```

### 使用 curl 手动测试

```bash
# 1. 登录
curl -X POST http://127.0.0.1:9123/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"userName":"Super","password":"123456"}'

# 2. 获取用户信息（需要替换 TOKEN）
curl -X GET http://127.0.0.1:9123/api/user/info \
  -H "Authorization: Bearer YOUR_TOKEN"

# 3. 更新个人信息
curl -X PUT http://127.0.0.1:9123/api/user/profile \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "realName": "张三",
    "email": "zhangsan@example.com",
    "phone": "13800138000"
  }'
```

## 常见问题

### 1. 后端端口被占用

如果 9123 端口被占用，可以修改 `backend/api/src/main.rs`:

```rust
.bind(("127.0.0.1", 9123))?  // 改为其他端口
```

同时需要修改前端代理配置 `frontend/.env.development`:

```
VITE_API_PROXY_URL=http://127.0.0.1:YOUR_PORT
```

### 2. 前端端口被占用

修改 `frontend/vite.config.ts`:

```typescript
server: {
  port: 4000,  // 改为其他端口
  // ...
}
```

### 3. 数据库文件位置

默认数据库文件位于 `backend/data.db`。

如需更改，设置环境变量：

```bash
export DATABASE_URL="sqlite://your_path/data.db?mode=rwc"
cargo run -p api
```

### 4. 重置数据库

删除数据库文件后重启后端即可：

```bash
cd backend
rm data.db
cargo run -p api
```

### 5. 前端代理错误

确保：
1. 后端服务已启动
2. `.env.development` 中的 `VITE_API_PROXY_URL` 正确
3. `.env` 中的 `VITE_ACCESS_MODE=backend`

### 6. 密码强度要求

新密码必须满足：
- 至少 8 位
- 包含字母和数字

示例有效密码: `Pass1234`, `Admin123`, `User2024`

## 开发建议

### 后端开发

1. **热重载**: 使用 `cargo-watch`
```bash
cargo install cargo-watch
cargo watch -x 'run -p api'
```

2. **代码检查**:
```bash
cargo check
cargo clippy
```

3. **格式化**:
```bash
cargo fmt
```

### 前端开发

1. **代码检查**:
```bash
pnpm lint
```

2. **格式化**:
```bash
pnpm lint:prettier
```

3. **类型检查**:
```bash
pnpm build  # 会先运行 vue-tsc
```

## 生产部署

### 后端

```bash
cd backend
cargo build --release
./target/release/api
```

### 前端

1. 修改 `.env.production` 中的 API 地址：
```
VITE_API_URL=https://your-api-domain.com
```

2. 构建：
```bash
cd frontend
pnpm build
```

3. 部署 `frontend/dist` 目录到静态服务器（Nginx、Apache 等）

## 技术支持

如遇到问题，请检查：
1. 后端日志输出
2. 前端浏览器控制台
3. 网络请求（浏览器开发者工具 Network 标签）

常见错误码：
- `401`: 未登录或 token 过期
- `403`: 无权限访问
- `404`: 资源不存在
- `500`: 服务器内部错误
