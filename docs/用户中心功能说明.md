# 用户中心功能说明

## 概述

本次更新实现了用户中心的完整功能，包括个人信息管理、密码修改和头像上传。前后端已完全对接。

## 新增功能

### 1. 个人信息管理

用户可以在用户中心页面编辑和更新个人信息。

#### 后端接口

**接口地址**: `PUT /api/user/profile`

**请求头**:
```
Authorization: Bearer {token}
Content-Type: application/json
```

**请求参数**:
```json
{
  "realName": "张三",           // 真实姓名（可选）
  "nickName": "超级管理员",      // 昵称（可选）
  "email": "user@example.com",  // 邮箱（可选）
  "phone": "13800138000",       // 手机号（可选）
  "gender": "男",               // 性别：男/女（可选）
  "address": "北京市朝阳区",     // 地址（可选）
  "bio": "个人简介"             // 个人简介（可选）
}
```

**响应示例**:
```json
{
  "code": 200,
  "msg": "更新成功",
  "data": {
    "userId": 1,
    "userName": "Super",
    "email": "user@example.com",
    "avatar": "/uploads/avatar_1.jpg",
    "nickName": "超级管理员",
    "phone": "13800138000",
    "gender": "男",
    "realName": "张三",
    "address": "北京市朝阳区",
    "bio": "个人简介",
    "roles": ["R_SUPER"],
    "buttons": ["UserAdd", "UserEdit", "UserDelete"]
  }
}
```

#### 前端实现

- 页面路径: `/system/user-center`
- 组件: `frontend/src/views/system/user-center/index.vue`
- API 调用: `fetchUpdateProfile(params)`

**功能特性**:
- 表单验证（必填项、格式校验）
- 编辑/保存模式切换
- 实时数据同步到 store
- 成功提示

### 2. 密码修改

用户可以修改自己的登录密码。

#### 后端接口

**接口地址**: `PUT /api/user/password`

**请求头**:
```
Authorization: Bearer {token}
Content-Type: application/json
```

**请求参数**:
```json
{
  "oldPassword": "123456",      // 当前密码（必填）
  "newPassword": "NewPass123"   // 新密码（必填，至少8位，需包含字母和数字）
}
```

**响应示例**:
```json
{
  "code": 200,
  "msg": "密码修改成功",
  "data": null
}
```

**错误响应**:
```json
{
  "code": 400,
  "msg": "当前密码不正确",
  "data": null
}
```

#### 前端实现

**功能特性**:
- 当前密码验证
- 新密码强度校验
- 确认密码一致性检查
- 修改成功后自动登出（可选）

### 3. 头像上传

用户可以上传和更新个人头像。

#### 后端接口

**接口地址**: `POST /api/user/avatar`

**请求头**:
```
Authorization: Bearer {token}
Content-Type: multipart/form-data
```

**请求参数**:
- `file`: 图片文件（支持 jpg, png, gif 等格式）

**响应示例**:
```json
{
  "code": 200,
  "msg": "头像上传成功",
  "data": {
    "userId": 1,
    "userName": "Super",
    "avatar": "/uploads/avatar_1_1733356800.jpg",
    ...
  }
}
```

#### 前端实现

**功能特性**:
- 点击头像触发文件选择
- 文件类型验证（仅图片）
- 文件大小限制（2MB）
- 上传成功后实时更新显示
- 错误提示

## 安全特性

### 密码安全

- **密码强度验证**: 至少 8 位，必须包含字母和数字
- **旧密码验证**: 修改密码时必须验证当前密码
- **密码哈希**: 使用 Argon2 算法存储密码
- **登录防暴力破解**: 5 次失败后锁定 5 分钟

### 文件上传安全

- **文件类型验证**: 仅允许图片格式
- **文件大小限制**: 前端限制 2MB
- **唯一文件名**: 使用 `avatar_{user_id}_{timestamp}.{ext}` 格式
- **存储隔离**: 文件存储在 `backend/uploads` 目录

## 前后端对接要点

### 1. 性别字段统一

- 后端存储: `"男"` 或 `"女"` (String)
- 前端显示: `"1"` (男) 或 `"2"` (女)
- 转换逻辑在前端处理

### 2. 用户信息同步

更新个人信息或头像后，前端需要：
1. 调用 API 更新
2. 更新 Pinia store 中的用户信息
3. 触发界面重新渲染

### 3. 密码修改流程

1. 用户输入当前密码和新密码
2. 前端验证新密码格式和一致性
3. 后端验证当前密码正确性
4. 后端验证新密码强度
5. 更新密码并返回成功
6. 前端提示用户重新登录（可选）

## 注意事项

1. **密码修改后**: 建议提示用户重新登录，当前 token 仍然有效
2. **头像路径**: 头像 URL 为相对路径，需要配合后端静态文件服务使用
3. **表单验证**: 前端已实现基本验证，后端也有相应验证
4. **错误处理**: 所有接口都有统一的错误响应格式

## 后续优化建议

1. **头像裁剪**: 添加前端图片裁剪功能
2. **头像压缩**: 后端自动压缩大图片

4. **操作日志**: 记录敏感操作（密码修改等）
